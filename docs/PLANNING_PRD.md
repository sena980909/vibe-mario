# [기획 PRD v2.0] Vibe Mario: 월드 익스팬션

## 1. 프로젝트 개요 (v2 업데이트)
- **비전:** 단순 모방을 넘어 웹 환경에 최적화된 독창적인 플랫폼 경험 제공.
- **추가 목표:** 사운드 피드백 강화, 다채로운 애니메이션, 로컬 스토리지 기반 진행도 저장.

## 2. 심화 시스템 요구사항

### 2.1 파워업 시스템 (Power-up System)
- **Small Mario:** 기본 상태. 적과 충돌 시 즉시 사망.
- **Super Mario (버섯):** 크기가 커지고 공격을 1회 견딤 (Small 상태로 복구). 머리 위 블록 파괴 가능.
- **Fire Mario (꽃):** 원거리 공격(불꽃 발사) 가능. (v2 추가 목표)

### 2.2 동적 엔티티 (Dynamic Entities)
- **움직이는 플랫폼:** 특정 경로를 왕복하거나 밟으면 아래로 떨어지는 발판.
- **비행 적:** 특정 높이에서 좌우로 비행하며 플레이어를 방해.
- **비밀 구역:** 특정 벽을 통과하거나 파이프를 통해 진입하는 보너스 맵.

### 2.3 오디오 및 이펙트 (Audio & SFX)
- **BGM:** 스테이지 분위기에 따른 3종 이상의 배경음악.
- **SFX:** 점프, 코인 획득, 적 처치, 파워업, 피격, 깃발 도달 시 효과음.
- **파티클:** 적 처치 시 먼지 효과, 블록 파괴 시 파편 효과.

## 3. UI/UX 고도화
- **일시정지 메뉴:** 게임 중 ESC 키로 접근. 설정(음량 조절) 및 메인으로 나가기.
- **로딩 스크린:** 자산(Asset) 로딩 바 및 게임 팁 출력.
- **점수 콤보:** 적을 연속으로 밟을 때 점수 배율 적용 (100 -> 200 -> 400...).

## 4. 레벨 및 월드 설계 (v2 상세)
- **월드 맵:** 스테이지를 선택할 수 있는 허브 월드 도입 고려.
- **수집 요소:** 각 스테이지당 '히든 스타(Hidden Star)' 3개 배치.

## 5. 데이터 저장
- **LocalStorage:** 클리어한 스테이지, 최고 점수, 수집한 스타 개수 저장.
